# NFS fstab 옵션 완전 정리

> 출처: https://blog.pages.kr/23

## 개요

`/etc/fstab` 파일은 "어떤 파일 시스템을 어디에 그리고 어떤 옵션을 가지고 마운트할 것인지"에 대한 정보를 담고 있습니다.

## NFS 마운트 기본 형식

```
server:/usr/local/pub    /pub   nfs    timeo=14,intr
```

---

## 주요 옵션

### 전송 크기 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| rsize=n | NFS 서버로부터 읽어들이는 바이트 수 | 1024 |
| wsize=n | NFS 서버에 쓰기할 때 사용하는 바이트 수 | 1024 |

### 타임아웃 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| timeo=n | RPC 타임아웃 시간 (1/10초 단위) | 7 |
| retrans=n | 주 타임아웃 발생 전 재전송 횟수 | 3 |

### 속성 캐시 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| acregmin=n | 일반 파일 속성 캐시 최소 시간 | 3초 |
| acregmax=n | 일반 파일 속성 캐시 최대 시간 | 60초 |
| acdirmin=n | 디렉토리 속성 캐시 최소 시간 | 30초 |
| acdirmax=n | 디렉토리 속성 캐시 최대 시간 | 60초 |
| actimeo=n | 위 네 옵션에 동시 적용 | - |

### 연결 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| retry=n | 백그라운드 마운트 재시도 횟수 | 10000 |
| port=n | NFS 서버 포트 번호 | 0 (portmapper) |
| mountport=n | mountd 포트 번호 | - |
| mounthost=name | mountd 실행 호스트명 | - |

### 마운트 방식 옵션
| 옵션 | 설명 |
|------|------|
| bg | 첫 시도 실패 시 백그라운드에서 계속 시도 |
| fg | 첫 시도 실패 시 즉시 포기 (기본값) |
| soft | 주 타임아웃 시 I/O 에러 보고 |
| hard | 주 타임아웃 시 무한 재시도 (기본값) |
| intr | 하드 마운트에서 신호를 통한 중단 허용 |

### 프로토콜 옵션
| 옵션 | 설명 |
|------|------|
| tcp | TCP 프로토콜 사용 |
| udp | UDP 프로토콜 사용 (기본값) |
| noac | 모든 속성 캐시 해제 |

## 1억 AI 도시 현재 설정

```bash
# /etc/fstab (73, 232 서버)
192.168.45.253:/home/ai /home/ai nfs defaults 0 0
```

### 권장 설정 (고성능)

```bash
192.168.45.253:/home/ai /home/ai nfs rsize=8192,wsize=8192,hard,intr 0 0
```

| 옵션 | 이유 |
|------|------|
| rsize=8192 | 읽기 성능 향상 |
| wsize=8192 | 쓰기 성능 향상 |
| hard | 연결 안정성 (끊김 방지) |
| intr | 필요시 중단 가능 |

---

## 옵션 부정형

Boolean 옵션은 `no` 접두사로 부정 가능:
```
nointr    # 파일 작업 인터럽트 불가
noac      # 속성 캐시 비활성화
```
