## 1. 클러스터

컴퓨팅 파워 증가를 위해 단일 고성능 컴퓨터의 한계를 극복하고자 "다수의 프로세서(CPU)가 하나의 문제를 협동적으로 계산하는 병렬컴퓨팅"이 등장했습니다.

**주요 개념:**
- **SMP**: 다수 프로세서가 하나의 메모리 공유
- **MPP**: 다수 프로세서가 각각 독립된 메모리 보유
- **NUMA**: 지역 메모리를 계층적으로 공유

최근 마이크로프로세서와 고속네트워크의 발전으로 단일 컴퓨터들을 네트워크로 연결하여 새로운 병렬컴퓨터를 구성할 수 있게 되었으며, 리눅스는 강력한 네트워크 성능과 자유로운 튜닝이 가능해 이상적인 OS입니다.

---

## 2. 클러스터의 필요성

**주요 장점:**
- 워크스테이션 대비 월등한 컴퓨팅 파워 제공
- 범용 하드웨어 사용으로 가격대 성능비 우수
- 노드 증설을 통한 자유로운 성능 향상
- 자체 제작 가능으로 문제 해결이 용이

---

## 3. 클러스터의 종류

### 3-1. 고계산용 클러스터 (HPC)

고성능 계산 능력을 제공하기 위해 제작되며, 주로 과학계산용으로 활용됩니다.

### 3-2. 부하분산 클러스터 (LVS)

대규모 서비스 제공을 위해 제작되며, 웹서비스에 활용가치가 높습니다.

**방식:**
- **NAT**: 로드밸런서가 모든 요청/응답 처리
- **IP Tunneling**: 응답이 로드밸런서를 거치지 않음 (리눅스 전용)
- **DR (Direct Routing)**: 최소한의 패킷 처리로 확장성 극대화

### 3-3. 고가용성 클러스터 (HA)

지속적인 서비스 제공을 목적으로 제작되며, 미션 크리티컬한 업무에 사용됩니다.

**주요 기술:**
- **heartbeat**: 로드밸런서와 백업서버 간 주기적 통신
- **fake**: 장애 발생 시 IP를 백업서버로 이주

### 3-4. 기타 클러스터 분류

**노드 소유권:**
- Dedicated Clusters
- Nondedicated Clusters

**하드웨어 기반:**
- Clusters of PCs (CoPs)
- Cluster of Workstations (COWs)
- Clusters of SMPs (CLUMPs)

**노드 구성:**
- **Homogeneous**: 동일 사양, 동일 OS
- **Heterogeneous**: 서로 다른 사양과 OS

## Building Cluster

### 1. 고려사항

#### H/W 고려사항

**프로세서 선택:**
- **ALPHA**: 부동소수점 연산 최강, 64bit 아키텍처
- **Intel**: 저가, 다양한 지원
- **AMD**: 저가, 벤치마크 우수

**네트워크:**
- 일반적으로 100Mbps 스위칭 허브 사용
- Myrinet, Gigabit Ethernet도 고려 가능

#### S/W 고려사항

**OS:** 리눅스가 가장 널리 사용됨

**MiddleWare:** 병렬라이브러리, 모니터링 툴, 큐잉시스템

### 2. 클러스터 구축하기

#### 구성도

4대의 컴퓨터(1개 관리노드 + 3개 컴퓨팅노드) 기본 구성

```
Master (관리노드) ─── Switch ─── Node2
                       │
                     Node3, Node4...
```

#### 2-1. 사설 네트워크 구축

**사설 네트워크 장점:**
- 관리 편리성 향상
- 보안 설정 단순화
- 외부 패킷 영향 제거
- 공인 IP 절약

#### 2-2. 관리노드 세팅

**네트워크 설정:**
```bash
# eth0 (외부 네트워크)
IP: 111.111.111.111

# eth1 (내부 네트워크)
IP: 192.168.1.1
Hostname: master.cluster.net
```

**hosts 파일 편집:**
```
vi /etc/hosts

210.123.54.230    master.cluster.net    master
192.168.1.1       node1.cluster.net     node1
192.168.1.2       node2.cluster.net     node2
192.168.1.3       node3.cluster.net     node3
192.168.1.4       node4.cluster.net     node4
```

**/home 공유 (NFS):**
```bash
vi /etc/exports

/home   node2 (rw,no_all_squash,no_root_squash)
/home   node3 (rw,no_all_squash,no_root_squash)
/home   node4 (rw,no_all_squash,no_root_squash)

# NFS 재시작
/etc/rc.d/init.d/nfs restart
```

#### 2-3. 컴퓨팅노드 세팅

**/home 마운트:**
```bash
vi /etc/fstab

192.168.1.1:/home   /home   nfs   hard,intr,rw
```

옵션 설명:
- `hard`: 강한 연결 (타임아웃 없음)
- `intr`: 프로세스 중단 가능
- `soft`: 약한 연결 (타임아웃 발생)

### 3. 보안

#### TCP Wrapper를 이용한 접근 제어

**모든 접근 차단:**
```bash
vi /etc/hosts.deny
ALL : ALL EXCEPT localhost : twist ( /etc/deny_message %h ) &
```

**허가된 IP 설정:**
```bash
vi /etc/hosts.allow
ALL : 127.0.0.1
ALL : 192.168.1.
```
