## 1. 주요 로그 파일 위치

```
/var/log/
├── syslog          # 시스템 전체 로그 (Ubuntu/Debian)
├── messages        # 시스템 전체 로그 (CentOS/RHEL)
├── auth.log        # 인증 로그 (SSH, sudo 등) ★
├── kern.log        # 커널 로그
├── dmesg           # 부팅 메시지
├── dpkg.log        # 패키지 설치 로그
├── fail2ban.log    # Fail2ban 차단 로그
├── nginx/          # 웹서버 로그
│   ├── access.log
│   └── error.log
└── pm2/            # PM2 로그
```

## 2. 로그 분석 명령어

### journalctl (systemd)
```bash
# 전체 로그
journalctl

# 최근 로그
journalctl -n 100

# 실시간 추적
journalctl -f

# 에러만
journalctl -p err

# 특정 서비스
journalctl -u nginx

# 시간 범위
journalctl --since "2025-12-27 00:00" --until "2025-12-27 23:59"

# 부팅 이후
journalctl -b
```

### grep 활용
```bash
# 에러 찾기
grep -i error /var/log/syslog

# SSH 실패
grep "Failed password" /var/log/auth.log

# 특정 IP
grep "192.168.45" /var/log/auth.log

# 최근 100줄에서
tail -100 /var/log/syslog | grep -i error
```

### 인증 로그 분석
```bash
# SSH 로그인 성공
grep "Accepted" /var/log/auth.log

# SSH 로그인 실패
grep "Failed password" /var/log/auth.log

# sudo 사용
grep "sudo:" /var/log/auth.log

# 무차별 공격 IP
grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10
```

## 3. 로그 자동 분석 스크립트

```bash
#!/bin/bash
# 일일 보안 로그 분석

echo "=== 보안 리포트 $(date) ==="

echo -e "\n[SSH 실패 TOP 10]"
grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10

echo -e "\n[sudo 사용]"
grep "sudo:" /var/log/auth.log | tail -10

echo -e "\n[시스템 에러]"
journalctl -p err --since "24 hours ago" | tail -20
```

## 4. ClamAV - 바이러스 스캔

### 설치
```bash
# Ubuntu/Debian
sudo apt install clamav clamav-daemon

# 바이러스 DB 업데이트
sudo freshclam
```

### 사용법
```bash
# 디렉토리 스캔
clamscan -r /home

# 감염 파일만 출력
clamscan -r -i /home

# 감염 파일 삭제
clamscan -r --remove /home

# 감염 파일 격리
clamscan -r --move=/quarantine /home

# 특정 확장자만
clamscan -r --include="\.php$" /var/www
```

### 자동화 (cron)
```bash
# 매일 새벽 2시 스캔
0 2 * * * /usr/bin/clamscan -r /home --log=/var/log/clamav/daily.log
```

## 5. rkhunter - 루트킷 탐지

### 설치
```bash
sudo apt install rkhunter
```

### 사용법
```bash
# DB 업데이트
sudo rkhunter --update

# 전체 검사
sudo rkhunter --check

# 경고만 표시
sudo rkhunter --check --rwo

# 자동 검사 (경고 없이)
sudo rkhunter --check --skip-keypress
```

### 화이트리스트 설정
```bash
# /etc/rkhunter.conf
SCRIPTWHITELIST=/usr/bin/egrep
ALLOWHIDDENDIR=/etc/.git
```

## 6. SSH Hub 구현 계획

### API 설계
```
GET /api/logs/errors     → 시스템 에러 로그
GET /api/logs/auth       → 인증 로그 (SSH 실패 등)
GET /api/logs/security   → 보안 이벤트
GET /api/scan/virus      → 바이러스 스캔 결과
GET /api/scan/rootkit    → 루트킷 스캔 결과
```

### NexusSSH 활용
```javascript
// 에러 로그 수집
const errors = await ssh.searchLog('/var/log/syslog', 'error', { ignoreCase: true, limit: 50 });

// SSH 실패 수집
const failures = await ssh.searchLog('/var/log/auth.log', 'Failed password', { limit: 100 });

// 바이러스 스캔 (백그라운드)
const scan = await ssh.runBackground('clamscan -r /home --log=/tmp/scan.log');
```

## 7. 주의사항

### ClamAV
```
- CPU/디스크 사용량 높음
- /dev, /proc 스캔 금지
- 실시간 스캔은 서버 부하 큼
```

### rkhunter
```
- 오탐(false positive) 많음
- 화이트리스트 설정 필수
- 정기적 업데이트 필요
```
