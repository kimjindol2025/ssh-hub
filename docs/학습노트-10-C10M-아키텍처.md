## C10K → C10M 진화

```
1999년 C10K:  10,000 동시 연결 (해결됨)
2010년 C1M:   1,000,000 동시 연결
2013년 C10M:  10,000,000 동시 연결
2025년 C100M: 100,000,000 동시 연결 (1억 AI 도시!)
```

---

## 핵심 원칙

### "커널이 문제다"
```
Robert Graham (2013):
"The kernel isn't the solution. The kernel is the problem."

문제:
- 컨텍스트 스위칭 오버헤드
- 락(Lock) 경합
- 메모리 복사 (커널 ↔ 유저스페이스)
- 인터럽트 처리
```

### 해결 방향
```
1. 커널 우회 (Kernel Bypass)
2. 락프리 자료구조
3. CPU 친화성 (Affinity)
4. 메모리 사전 할당
5. 배치 처리
```

## 아키텍처 설계

### 파티셔닝
```
┌────────────────────────────────────────────┐
│                  서버                       │
├─────────────────┬──────────────────────────┤
│   Control Plane │      Data Plane          │
│   (OS 관리)      │   (커널 우회)            │
├─────────────────┼──────────────────────────┤
│  CPU 0-1        │   CPU 2-15               │
│  일반 OS 작업    │   네트워크 처리 전용       │
└─────────────────┴──────────────────────────┘
```

### 커널 우회 기술

| 기술 | 설명 |
|------|------|
| **DPDK** | Intel 데이터 플레인 개발 키트 |
| **Netmap** | FreeBSD/Linux 고속 패킷 I/O |
| **XDP** | eBPF 기반 고속 패킷 처리 |
| **io_uring** | 비동기 I/O (Linux 5.1+) |

## 필수 튜닝

### 시스템 설정
```bash
# /etc/sysctl.d/99-c10m.conf

# 파일 핸들
fs.file-max = 100000000
fs.nr_open = 100000000

# 네트워크
net.core.somaxconn = 65535
net.core.netdev_max_backlog = 65536
net.ipv4.tcp_max_syn_backlog = 65536

# TCP
net.ipv4.ip_local_port_range = 1024 65535
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 10

# 메모리
net.core.rmem_max = 67108864
net.core.wmem_max = 67108864
```

### 메모리 계산
```
1억 연결 × 3KB/연결 = 300GB

분산:
- 10서버 × 1000만 연결 × 3KB = 30GB/서버
- 현실적으로 가능!
```
