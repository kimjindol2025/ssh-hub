
## 1. 로드 밸런싱

### 전략: 가중치 기반 + 실시간 부하

```javascript
// 서버 선택 알고리즘
function selectServer(servers) {
    // 1. 부하 수집 (CPU, 메모리, 연결 수)
    // 2. 점수 계산 (낮을수록 좋음)
    // 3. 가장 여유로운 서버 반환

    return servers
        .filter(s => s.status === 'online')
        .filter(s => s.load < 80)  // 80% 이상은 제외
        .sort((a, b) => a.score - b.score)[0];
}
```

### 부하 점수 계산

```
점수 = (CPU% × 0.4) + (MEM% × 0.3) + (연결수/최대 × 0.3)

예시:
- 73:  (30×0.4) + (40×0.3) + (1000/5000×0.3) = 12 + 12 + 6 = 30점
- 232: (60×0.4) + (70×0.3) + (3000/5000×0.3) = 24 + 21 + 18 = 63점
- 253: (20×0.4) + (30×0.3) + (500/5000×0.3) = 8 + 9 + 3 = 20점 ✓ 선택
```

## 2. 자동 페일오버

### 시나리오

```
정상 상태:
AI-12345 ←→ 232 서버

232 서버 다운:
1. Hub가 감지 (1초 이내)
2. AI-12345 세션 → 73 서버로 전환
3. AI는 끊김 없이 계속 작업

232 서버 과부하 (90%):
1. Hub가 감지
2. 신규 연결 → 다른 서버로
3. 기존 연결 → 점진적 마이그레이션
```

### 헬스체크

```javascript
// 1초마다 모든 서버 체크
setInterval(async () => {
    for (const server of SERVERS) {
        const health = await checkHealth(server);

        if (!health.alive) {
            // 즉시 페일오버
            migrateAllSessions(server, getAlternative());
        } else if (health.load > 90) {
            // 점진적 분산
            redistributeSessions(server, 20); // 20%씩 이동
        }
    }
}, 1000);
```

## 3. 공유 환경

### NFS 구성

```
┌─────────────────────────────────────────┐
│         NFS 서버 (73 또는 별도)          │
│                                         │
│  /exports/ai-homes/                     │
│  ├── ai-00000001/                       │
│  ├── ai-00000002/                       │
│  ├── ...                                │
│  └── ai-99999999/                       │
└─────────────────────────────────────────┘
           │
    ┌──────┴──────┐
    ▼             ▼
  73서버        232서버       253서버
  mount         mount         mount
  /home/ai      /home/ai      /home/ai
```

### 이점

```
AI가 73에서 작업 → 파일 저장
AI가 232로 이동 → 같은 파일 있음
= 끊김 없는 경험
```

## 4. 무한 확장

### 서버 추가 절차

```bash
# 1. 새 서버 준비 (10.0.0.100)
ssh root@10.0.0.100

# 2. NFS 마운트
mount -t nfs 73:/exports/ai-homes /home/ai

# 3. SSH Hub에 등록
curl -X POST ssh.dclub.kr/api/servers \
  -d '{"id": "100", "host": "10.0.0.100", "port": 22}'

# 4. 자동으로 로드밸런싱에 포함
# 끝. 1분 만에 확장 완료.
```

### 확장 계획

```
현재: 3대 (73, 232, 253)
├── 1억 AI ÷ 3 = 서버당 3,300만
├── 너무 많음

목표: 서버당 100만 AI
├── 1억 ÷ 100만 = 100대 필요
├── 클라우드 자동 스케일링 연동 가능
```

## 7. API 설계

### 서버 관리

```
GET  /api/cluster/status     전체 클러스터 상태
GET  /api/cluster/servers    서버 목록 + 부하
POST /api/cluster/add        서버 추가
DEL  /api/cluster/remove/:id 서버 제거
```

### 세션 관리

```
GET  /api/sessions           전체 세션 목록
GET  /api/sessions/:ai       AI별 세션
POST /api/sessions/migrate   세션 마이그레이션
```

### AI용 (토큰 최소)

```
GET  /api/ai/cluster   → "3servers,1억ai,avg:45%"
GET  /api/ai/my/:id    → "server:253,uptime:2d"
```
