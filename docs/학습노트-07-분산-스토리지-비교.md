# NFS vs GlusterFS vs CephFS 분산 스토리지 비교

> 학습일: 2025-12-27
> 3번 Claude 학습노트 #07

---

## 비교 요약

| 항목 | NFS | GlusterFS | CephFS |
|------|-----|-----------|--------|
| **유형** | 네트워크 파일시스템 | 블록 기반 분산 | 객체 기반 분산 |
| **확장성** | 제한적 | 페타바이트 | 페타바이트+ |
| **복잡도** | 단순 | 중간 | 높음 |
| **적합 용도** | 소규모 공유 | 대용량 순차 접근 | 빈번한 랜덤 접근 |
| **메타데이터** | 서버 의존 | 분산 해싱 | 분산 MDS |

---

## NFS (현재 사용 중)

### 특징
```
- 전통적인 네트워크 파일시스템
- 단일 서버 (SPOF 위험)
- 간단한 설정
- NFSv4: 보안 강화
```

### 1억 AI 도시 현재 설정
```
253 (서버) → /home/ai 공유
    ↓
73, 232 (클라이언트) → /home/ai 마운트
```

### 장점/단점
```
✅ 설정 간단
✅ 즉시 사용 가능
❌ 서버 다운 시 전체 중단
❌ 확장 한계
```

---

## GlusterFS

### 특징
```
- 메타데이터 서버 없음 (해싱 방식)
- 블록 크기: 128KB (기본)
- 페타바이트 확장
- 대용량 파일에 적합
```

### 아키텍처
```
┌─────────────────────────────────────┐
│         GlusterFS 클러스터          │
├──────────┬──────────┬───────────────┤
│  Brick1  │  Brick2  │    Brick3     │
│   (73)   │  (232)   │    (253)      │
└──────────┴──────────┴───────────────┘
        ↑
    Elastic Hash (DHT)
```

### 볼륨 유형
```
Distributed: 파일 분산 저장
Replicated:  복제 (HA)
Striped:     파일 분할 (성능)
Dispersed:   Erasure Coding (효율)
```

### 설치 예시
```bash
# 모든 서버
apt install glusterfs-server

# 73에서 클러스터 구성
gluster peer probe 192.168.45.232
gluster peer probe 192.168.45.253

# 볼륨 생성 (복제)
gluster volume create ai-homes replica 3 \
    192.168.45.73:/data/brick \
    192.168.45.232:/data/brick \
    192.168.45.253:/data/brick

gluster volume start ai-homes
```

## CephFS

### 특징
```
- 객체 기반 스토리지
- RADOS 백엔드
- 블록(RBD) + 파일 + 객체 통합
- 강력한 일관성
```

### 아키텍처
```
┌────────────────────────────────────────┐
│              CephFS                    │
├────────────────────────────────────────┤
│  MDS (Metadata Server)                 │
├────────────────────────────────────────┤
│              RADOS                     │
├──────────┬──────────┬──────────────────┤
│   OSD    │   OSD    │      OSD         │
│  (73)    │  (232)   │     (253)        │
├──────────┴──────────┴──────────────────┤
│             MON (Monitor)              │
└────────────────────────────────────────┘
```

### 구성 요소
```
MON: 클러스터 상태 관리 (최소 3개)
OSD: 실제 데이터 저장 (Object Storage Daemon)
MDS: 메타데이터 서버 (CephFS용)
```

## 성능 비교

### 소규모 파일 (4KB)
```
NFS:      ████████████████ 100%
CephFS:   ██████████████   87%
GlusterFS: ████████        50% (삭제 시 느림)
```

### 대용량 파일 (1GB+)
```
GlusterFS: ████████████████ 100%
CephFS:    ██████████████   85%
NFS:       ████████         50%
```

## 1억 AI 도시 권장

### 현재 (3서버)
```
NFS 유지 (충분)
- 설정 완료됨
- 안정적
- 복잡도 낮음
```

### 확장 시 (10서버+)
```
GlusterFS 고려
- 서버 추가 용이
- NFS보다 HA 우수
- AI 홈 디렉토리 분산
```

### 대규모 (100서버+)
```
Ceph 필수
- 오브젝트 스토리지
- 블록 스토리지
- 완전 분산
```
